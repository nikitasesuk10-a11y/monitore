# Описание проекта

Данное приложение представляет собой простой системный монитор, разработанный на Python. Оно отображает основную информацию о состоянии компьютера в режиме реального времени: загрузку процессора, использование оперативной памяти, дискового пространства и сетевой активности. Проект был выполнен в рамках изучения объектно-ориентированного программирования.

# Структура программы

Приложение состоит из двух основных файлов:

* **metrics.py** — содержит классы, отвечающие за получение и обработку информации о системных ресурсах.
* **main.py** — реализует графический интерфейс и механизм периодического обновления данных.

## Реализация метрик

В файле `metrics.py` используется абстрактный базовый класс `Metric`, который задаёт общий интерфейс для всех типов метрик.

От него наследуются следующие классы:

* `CPUMetric` — получает текущую загрузку процессора и его частоту;
* `RAMMetric` — отображает объём используемой оперативной памяти;
* `DiskMetric` — показывает степень заполнения диска;
* `NetworkMetric` — собирает информацию о количестве переданных и полученных данных по сети.

Для получения системной информации используется библиотека `psutil`.

## Использование полиморфизма

Базовый класс `Metric` содержит абстрактные методы `update()` и `get_info()`, которые переопределяются в дочерних классах. Благодаря этому основной код работает с объектами через единый интерфейс и не зависит от конкретного типа метрики. При обновлении данных достаточно вызвать методы `update()` и `get_info()`, независимо от того, к какому ресурсу относится объект.

## Использование инкапсуляции

Данные внутри классов хранятся в защищённых атрибутах. Для доступа к основным параметрам используются свойства (`@property`). Такой подход позволяет скрыть детали реализации и предоставить только необходимый интерфейс для работы с объектами.

# Графический интерфейс

Интерфейс приложения создан с помощью библиотеки Flet.

Основная логика сосредоточена в классе `MonitorApp`:

* метод `_build_ui()` формирует элементы интерфейса для отображения каждой метрики;
* все созданные объекты сохраняются в коллекции `self.metrics`;
* метод `_start_update()` запускает отдельный поток выполнения, который отвечает за регулярное обновление данных;
* метод `_refresh()` проходит по списку метрик и вызывает их обновление.

Информация на экране обновляется каждые две секунды.

# Запуск программы

Для запуска из исходного кода необходимо выполнить команду:

```bash
python main.py
```

Также доступна готовая исполняемая версия приложения (`MonitorApp.exe`), которую можно скачать в разделе Releases репозитория проекта.

# Используемые библиотеки

* **Flet** — создание графического интерфейса;
* **psutil** — получение информации о состоянии системы и её ресурсах.
