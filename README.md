# Описание

Простое десктопное приложение на Python, которое показывает загрузку компьютера в реальном времени: процессор, оперативная память, диск и сеть. Сделано для зачета по ООП.

## Логика

Вся программа состоит из двух файлов:

- `metrics.py` — описаны классы, которые умеют получать данные о ресурсах компьютера.
- `main.py` — собирается интерфейс (окно, прогресс-бары, текст) и всё это крутится в цикле, обновляясь каждые 2 секунды.

### metrics.py — классы метрик

Тут у меня есть один базовый абстрактный класс `Metric`, от которого наследуются все конкретные метрики:

- `CPUMetric` — процент загрузки процессора и его частота
- `RAMMetric` — сколько занято оперативной памяти
- `DiskMetric` — сколько занято места на диске
- `NetworkMetric` — сколько байт отправлено/получено по сети за время работы программы

Все данные берутся через библиотеку `psutil`


### Полиморфизм

В базовом классе `Metric` я объявил два абстрактных метода: `update()` (обновить значение) и `get_info()`
код в `main.py` вызывает `metric.update()` и `metric.get_info()`, не зная и не заботясь о том, какая именно метрика перед ним — CPU, RAM, диск или сеть.

### Инкапсуляция

Поля у классов сделал "приватными" (через `_`), а доступ к ним — через свойства (`@property`). Наружу торчат только `name` и `value`, а как они считаются — дело самого класса.

### main.py — интерфейс

все на `flet` — 

Класс `MonitorApp` всё в себе и держит:

1. В `_build_ui()` создаются карточки для каждой метрики (прогресс-бар + текст), и складываются в список `self.metrics`. 
2. В `_start_update()` запускается отдельный поток (`threading.Thread`), который каждые 2 секунды вызывает `_refresh()`. 
3. `_refresh()` пробегает по списку метрик и для каждой вызывает `metric.update()`

## Как запустить

```
python main.py
```

- `psutil` — для получения данных о системе
